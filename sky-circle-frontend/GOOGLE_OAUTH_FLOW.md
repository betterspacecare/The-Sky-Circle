# Google OAuth Flow Diagram

## 🔄 Authentication Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                     User Authentication Flow                     │
└─────────────────────────────────────────────────────────────────┘

1. USER CLICKS "CONTINUE WITH GOOGLE"
   │
   ├─ Login Page (/login)
   │  └─ handleGoogleLogin() called
   │
   └─ Signup Page (/signup)
      └─ handleGoogleSignup() called

2. SUPABASE AUTH INITIATED
   │
   └─ supabase.auth.signInWithOAuth({
        provider: 'google',
        options: {
          redirectTo: 'http://localhost:3000/auth/callback'
        }
      })

3. REDIRECT TO GOOGLE
   │
   └─ User sees Google sign-in page
      ├─ Select Google account
      ├─ Review permissions
      └─ Click "Continue"

4. GOOGLE AUTHENTICATES
   │
   └─ Google validates user
      └─ Generates authorization code

5. REDIRECT TO SUPABASE
   │
   └─ https://[project-id].supabase.co/auth/v1/callback?code=xxx
      └─ Supabase exchanges code for session

6. REDIRECT TO YOUR APP
   │
   └─ http://localhost:3000/auth/callback?code=xxx
      └─ Auth callback handler processes

7. CALLBACK HANDLER (/app/auth/callback/route.ts)
   │
   ├─ Exchange code for session
   ├─ Check if user profile exists
   │
   ├─ NEW USER:
   │  ├─ Create user profile in database
   │  ├─ Set display_name from Google
   │  ├─ Set profile_photo_url from Google
   │  ├─ Generate referral_code
   │  └─ Redirect to /setup-profile
   │
   └─ EXISTING USER:
      └─ Redirect to /dashboard

8. USER LOGGED IN ✅
   │
   └─ Session stored in cookies
      └─ User can access protected routes
```

## 📊 Data Flow

```
Google Account Data → Supabase → Your Database
─────────────────────────────────────────────

From Google:
├─ email              → auth.users.email
├─ full_name          → users.display_name
├─ avatar_url         → users.profile_photo_url
└─ email_verified     → auth.users.email_confirmed_at

Generated by App:
├─ referral_code      → users.referral_code (8 chars)
├─ total_points       → users.total_points (default: 0)
└─ level              → users.level (default: 1)
```

## 🔐 Security Flow

```
┌─────────────────────────────────────────────────────────┐
│                   Security Layers                        │
└─────────────────────────────────────────────────────────┘

1. HTTPS Only (Production)
   └─ All OAuth traffic encrypted

2. State Parameter
   └─ Prevents CSRF attacks
   └─ Automatically handled by Supabase

3. Authorization Code
   └─ One-time use code
   └─ Exchanged for access token
   └─ Short expiration time

4. Access Token
   └─ Stored securely in httpOnly cookies
   └─ Not accessible via JavaScript
   └─ Auto-refreshed by Supabase

5. Session Management
   └─ Server-side session validation
   └─ Automatic token refresh
   └─ Secure logout
```

## 🔄 Token Refresh Flow

```
┌─────────────────────────────────────────────────────────┐
│              Automatic Token Refresh                     │
└─────────────────────────────────────────────────────────┘

Access Token Expires (1 hour)
       │
       ▼
Supabase Client Detects Expiration
       │
       ▼
Automatically Calls Refresh Endpoint
       │
       ▼
Uses Refresh Token (stored in httpOnly cookie)
       │
       ▼
Gets New Access Token
       │
       ▼
Updates Session
       │
       ▼
User Stays Logged In ✅

Note: Refresh token valid for 30 days
After 30 days, user must sign in again
```

## 🚨 Error Handling Flow

```
┌─────────────────────────────────────────────────────────┐
│                   Error Scenarios                        │
└─────────────────────────────────────────────────────────┘

User Cancels Google Sign-in
       │
       └─→ Redirect back to login page
           └─→ No error shown (user choice)

Invalid Credentials
       │
       └─→ Show error message
           └─→ "Failed to sign in with Google"

Network Error
       │
       └─→ Show error message
           └─→ "Connection failed, please try again"

Profile Creation Fails
       │
       └─→ Log error to console
           └─→ Show generic error
               └─→ User can retry

Session Expired
       │
       └─→ Automatic refresh attempt
           │
           ├─→ Success: Continue
           └─→ Fail: Redirect to login
```

## 📊 Database Schema

```sql
-- Users table (created automatically)
CREATE TABLE users (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  email TEXT NOT NULL,
  display_name TEXT,
  profile_photo_url TEXT,
  referral_code TEXT UNIQUE,
  total_points INTEGER DEFAULT 0,
  level INTEGER DEFAULT 1,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Auth users (managed by Supabase)
auth.users (
  id UUID,
  email TEXT,
  email_confirmed_at TIMESTAMP,
  user_metadata JSONB,  -- Contains Google data
  created_at TIMESTAMP
)
```
